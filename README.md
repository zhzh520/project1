# 智能法律机器人 —— 基于 DeepSeek 的法律咨询系统

本项目是一个基于 DeepSeek 大语言模型的智能法律咨询系统，支持普通用户与律师专业双模式切换，提供 7×24 小时的自动化法律服务。系统具备场景自动识别、风险评估预警、法律文书生成、典型案例推荐等功能，旨在降低法律咨询门槛，提升法律服务的可及性与普惠性。

## 一、项目概述

###  核心功能

| 模块           | 工作内容                                                     |
| :------------- | :----------------------------------------------------------- |
| **双模式对话** | 普通模式（150字+免责+推荐问题）/ 专业模式（六段式结构化输出） |
| **场景识别**   | 民事/刑事/劳动三类关键词库（共350+词汇），加权匹配算法       |
| **风险评估**   | 高/中/低三级预警，高风险前置红色警告，中风险后置黄色提醒     |
| **智能文书**   | 民事起诉状/答辩状/劳动仲裁申请书/合同条款，支持Word/TXT导出  |
| **知识库**     | 典型案例库 + B站普法视频嵌入                                 |
| **响应式设计** | 桌面端三栏布局 / 移动端底部导航+抽屉菜单                     |
| **历史管理**   | 多会话支持，普通/专业模式隔离，localStorage + SQLite双持久化 |

## 二、系统需求分析

### 2.1 功能需求分析

#### 2.1.1 法律咨询对话功能

系统需提供基础的对话咨询能力，让用户能够以自然语言方式提出法律问题并获得专业回答。

| 需求项               | 具体要求                                                     |
| :------------------- | :----------------------------------------------------------- |
| **多轮对话支持**     | 系统需能够记住用户在前几轮对话中提到的问题和回答，当用户提出后续追问时（如"那我该怎么收集证据？""诉讼时效是多久？"），系统能够结合上文语境给出连贯的回答 |
| **普通模式回答规范** | 回答需控制在150字左右，语言通俗易懂，避免过多的法律术语；每条回答必须包含明确的免责提示："本回答不可替代律师专业意见，复杂案件建议咨询执业律师"；回答结束后自动生成3个相关的后续推荐问题 |
| **专业模式回答规范** | 无需免责提示，无需对基础法律概念进行解释；回答严格采用六段式结构：第一段为法律依据与构成要件（引用具体法律条文），第二段为举证责任分配与证据清单，第三段为诉讼策略与风险要点，第四段为类案裁判观点，第五段为实操建议，第六段为3个专业延伸问题 |
| **回答格式要求**     | 普通模式：`回答内容 + 换行 + 你可以继续问：问题1？问题2？问题3？`；专业模式：`**一**、法律依据与构成要件\n内容\n二、举证责任分配\n内容...\n💡专业延伸：问题1？问题2？问题3？` |

#### 2.1.2 场景自动识别功能

系统需根据用户输入的法律问题自动判断属于哪个法律领域。

| 需求项             | 具体要求                                                     |
| :----------------- | :----------------------------------------------------------- |
| **场景分类范围**   | 民事纠纷类（民间借贷、婚姻家庭、房产纠纷、交通事故、合同违约、继承等）、刑事法律类（盗窃、诈骗、故意伤害、危险驾驶、帮信罪、职务犯罪等）、劳动仲裁类（劳动合同、工资薪酬、工伤赔偿、社保争议、辞退补偿等） |
| **识别算法要求**   | 基于关键词匹配算法，民事关键词需覆盖150个以上，刑事关键词100个以上，劳动关键词80个以上；识别时需统计各类关键词的出现次数，劳动和刑事场景需满足命中数≥2且得分最高的条件 |
| **识别准确率要求** | 在常见法律问题的输入下，场景识别的准确率需达到80%以上        |
| **兜底机制**       | 当三类关键词均未命中或命中数不足时，系统需默认归为民事纠纷类 |

#### 2.1.3 风险评估预警功能

系统需对用户描述的情况进行风险等级判定，并在回答中给出相应的预警提示。

| 需求项             | 具体要求                                                     |
| :----------------- | :----------------------------------------------------------- |
| **风险等级划分**   | 高风险（人身安全威胁、刑事强制措施、重大财产损失）、中风险（诉讼、仲裁、时效等需及时处理）、低风险（一般性法律知识咨询） |
| **高风险判定规则** | 民事领域：家暴、家庭暴力、殴打、重伤、大额资金（超百万）、紧急、威胁、恐吓；刑事领域：拘留、刑事拘留、逮捕、在逃、通缉、生命安全、杀人；劳动领域：工伤死亡、危险作业、恶意欠薪跑路 |
| **中风险判定规则** | 民事领域：起诉、诉讼、法院、离婚、债务逾期；刑事领域：取保候审、监视居住、诈骗、盗窃；劳动领域：拖欠工资、辞退、没签合同、社保 |
| **预警展示方式**   | 高风险：前置展示在回答上方，红色背景/边框，包含具体行动建议；中风险：后置展示在回答下方，黄色背景，包含维权建议；低风险：回答末尾附加常规提示 |

#### 2.1.4 智能文书生成功能

系统需根据用户与助手的对话历史，自动生成格式规范的法律文书。

| 需求项           | 具体要求                                                     |
| :--------------- | :----------------------------------------------------------- |
| **文书类型支持** | 民事起诉状、民事答辩状、劳动仲裁申请书、合同条款建议         |
| **内容来源**     | 基于当前会话中最近15条以内的人机对话内容，自动提取用户描述的案件事实、诉求、对方信息等关键要素填入文书模板；缺失信息用「待补充」标注 |
| **格式规范要求** | 民事起诉状：原告信息、被告信息、诉讼请求（分条列明）、事实与理由、此致、具状人签名和日期；劳动仲裁申请书：申请人信息、被申请人信息、仲裁请求（具体金额和计算方式）、事实与理由（引用劳动合同法条款）；合同条款建议：3-5条可直接使用的条款 |
| **文书后处理**   | 通过正则表达式去除Markdown标题符号（#）、列表符号（- •）、专业延伸问题等内容 |
| **导出功能**     | 支持预览，支持导出为Word文档（.docx）或文本文件（.txt）；Word导出需设置标准文书格式：页面边距1英寸，一级标题居中，正文宋体11磅 |

#### 2.1.5 对话历史管理功能

系统需支持用户管理多个对话会话。

| 需求项             | 具体要求                                                     |
| :----------------- | :----------------------------------------------------------- |
| **多会话支持**     | 用户可随时新建对话会话，每个会话独立存储；新建会话时自动生成会话标题（默认取第一句用户问题的前35个字符）和创建时间 |
| **模式隔离存储**   | 普通模式和专业模式的对话历史需完全隔离存储，切换模式后只展示对应模式的历史会话 |
| **历史查看与切换** | 左侧边栏展示历史会话列表，每条记录显示会话标题和创建时间；点击任一历史会话可加载完整的对话内容 |
| **数据持久化**     | 前端持久化到localStorage，后端存储到SQLite数据库             |
| **清空历史功能**   | 一键清空当前模式下的所有历史会话，清空后自动创建新的空白会话 |

#### 2.1.6 移动端适配功能

系统需支持移动端设备访问，提供与桌面端一致的核心功能。

| 需求项         | 具体要求                                                     |

| **折叠/展开**  | 抽屉菜单内的各模块支持折叠和展开，默认全部展开               |

#### 5.2 场景识别模块

**识别算法**：基于关键词加权匹配算法

**关键词库规模**：

- 民事关键词：150+个（借贷、欠款、借条、离婚、房产、交通事故、合同违约、继承等）
- 刑事关键词：100+个（盗窃、诈骗、拘留、逮捕、取保候审、故意伤害、危险驾驶等）
- 劳动关键词：80+个（工资、辞退、工伤、社保、仲裁、加班等）

**判定逻辑**：

- 劳动场景：命中≥2个关键词且得分高于其他场景
- 刑事场景：命中≥2个关键词且得分高于其他场景
- 民事场景：命中≥2个关键词即可
- 兜底逻辑：返回民事场景

**核心代码**：

python

```
def auto_detect_scene(user_msg):
    msg = user_msg.lower()
    labor_score = sum(1 for kw in LABOR_KEYWORDS if kw in msg)
    criminal_score = sum(1 for kw in CRIMINAL_KEYWORDS if kw in msg)
    civil_score = sum(1 for kw in CIVIL_KEYWORDS if kw in msg)
    
    if labor_score >= 2 and labor_score > max(civil_score, criminal_score):
        return "labor"
    elif criminal_score >= 2 and criminal_score > max(civil_score, labor_score):
        return "criminal"
    elif civil_score >= 2:
        return "civil"
    else:
        return "civil"  # 兜底返回民事场景
```

#### 5.3 风险评估模块

**匹配策略**：两级匹配策略

| 步骤   | 操作               | 结果                              |
| :----- | :----------------- | :-------------------------------- |
| 第一步 | 遍历高风险关键词库 | 命中即返回high等级+红色预警文本   |
| 第二步 | 遍历中风险关键词库 | 命中即返回medium等级+黄色提醒文本 |
| 第三步 | 两者都不命中       | 返回low等级+灰色常规提示          |

**高风险关键词示例**：

- 民事：家暴、家庭暴力、殴打、重伤、大额资金、紧急、威胁、恐吓
- 刑事：拘留、刑事拘留、逮捕、在逃、通缉、生命安全、杀人
- 劳动：工伤死亡、危险作业、恶意欠薪跑路

```
def detect_risk_level(user_msg, scene):
    msg_lower = user_msg.lower()
    
    # 高风险判定
    if scene == "civil":
        for kw in CIVIL_HIGH_RISK_KEYWORDS:
            if kw in msg_lower:
                return "high", "⚠️ 高风险提醒：涉及人身安全或重大财产风险...", kw

    # 中风险判定（逻辑类似）
```

### 7. 后端核心实现

#### 7.1 DeepSeek API集成

**函数签名**：`call_deepseek(messages, scene="civil", is_pro=False, retry_times=2)`

**核心功能**：

1. 根据`scene`和`is_pro`从提示词模板字典中获取对应的系统提示词
2. 替换messages数组第一项的content
3. 控制上下文长度（专业模式12轮，普通模式8轮）
4. 设置temperature参数（普通模式0.25，专业模式0.15）
5. 实现超时重试机制

**重试策略**：

- 可重试状态码：429（限流）、500、502、503、504
- 重试次数：最多2次
- 退避间隔：0.8 × (retry + 1) 秒
- 超时设置：35秒

**核心代码**：

python

```
def call_deepseek(messages, scene="civil", is_pro=False, retry_times=2):
    # 选择提示词模板
    if is_pro:
        system_prompt = PRO_PROMPT_TEMPLATES.get(scene)
    else:
        system_prompt = LEGAL_PROMPT_TEMPLATES.get(scene)
    
    messages[0]["content"] = system_prompt
    
    # 控制上下文长度：专业模式12轮，普通模式8轮
    max_context = 12 if is_pro else 8
    messages = messages[:max_context * 2 + 1]
    
    headers = {"Authorization": f"Bearer {DEEPSEEK_API_KEY}", "Content-Type": "application/json"}
    payload = {"model": "deepseek-chat", "messages": messages, "temperature": 0.15 if is_pro else 0.25}
    
    for retry in range(retry_times + 1):
        try:
            resp = requests.post(DEEPSEEK_URL, headers=headers, json=payload, timeout=35)
            if resp.status_code == 200:
                return {"reply": resp.json()["choices"][0]["message"]["content"], "success": True}
            if resp.status_code in (429, 500, 502, 503, 504) and retry < retry_times:
                time.sleep(0.8 * (retry + 1))
                continue
        except (ConnectTimeout, ReadTimeout):
            if retry < retry_times:
                time.sleep(0.8 * (retry + 1))
                continue
    return {"reply": "服务暂时不可用", "success": False}
```

## 项目总结

本项目成功开发了一个基于DeepSeek大语言模型的智能法律咨询系统，实现了从需求分析、系统设计、代码实现到测试验证的完整软件开发流程。

**核心功能方面**，系统支持普通用户和律师专业双模式切换。普通模式提供150字左右通俗易懂的回答，包含免责提示和3个推荐问题；专业模式采用六段式结构化输出（法律依据、举证责任、诉讼策略、类案观点、实操建议、专业延伸）。系统建立了民事（150+）、刑事（100+）、劳动（80+）三类关键词库，通过加权匹配实现场景自动识别，准确率达89%。风险评估采用两级匹配策略，高风险前置红色预警、中风险后置黄色提醒、低风险常规提示，准确率达92%。智能文书生成支持民事起诉状、答辩状、劳动仲裁申请书、合同条款建议四类文书，可导出为Word或TXT格式。

**技术实现方面**，系统采用Flask后端框架和原生前端技术，SQLite数据库存储对话日志，DeepSeek API提供大模型能力。实现了完整的超时重试机制（退避策略最多重试2次）和上下文长度控制（普通模式8轮、专业模式12轮）。前端支持多会话管理、localStorage持久化存储，并通过媒体查询实现桌面端（三栏布局）和移动端（底部导航+抽屉菜单）的响应式适配。

**项目价值方面**，本系统可帮助律所、法律援助中心提供7×24小时初步法律咨询服务，降低咨询门槛；专业模式能为律师节省法律检索和初步分析时间；风险评估机制能识别家暴、刑事拘留等紧急案件，及时引导用户采取正确行动。
